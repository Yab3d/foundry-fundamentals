========================================================
SOLIDITY DECIMAL MATH EXPLANATION
========================================================

Topic:
Why we divide by 1e18 in ETH → USD conversion

Code Reference:

ethAmountInUsd = (ethPrice * ethAmount) / 1e18;


--------------------------------------------------------
WHAT IS 1e18?
--------------------------------------------------------

1e18 means:

1 × 10^18

Which equals:

1000000000000000000


Ethereum uses 18 decimal precision because:

1 ETH = 10^18 Wei


--------------------------------------------------------
THE DECIMAL EXPLOSION PROBLEM
--------------------------------------------------------

In Solidity:

ETH Price from Chainlink is returned with 18 decimals.

ETH Amount sent by user is also stored with 18 decimals.

So when multiplying:

ethPrice * ethAmount

Decimals are doubled.

Example:

ETH Price:

2000 USD = 2000 × 10^18
= 2000000000000000000000


ETH Amount:

0.002 ETH
= 0.002 × 10^18
= 2000000000000000


Multiplication:

(2000 × 10^18) × (0.002 × 10^18)

= 2000 × 0.002 × 10^36

Now we have:

36 decimal places ❌
Too many decimals for Solidity calculations.


--------------------------------------------------------
WHY DIVIDE BY 1e18?
--------------------------------------------------------

Division by 1e18 normalizes the decimals.

Because:

10^36 / 10^18 = 10^18

So we return to Solidity standard precision.


Final Formula:

ethAmountInUsd =
(ethPrice * ethAmount) / 1e18


--------------------------------------------------------
REAL EXAMPLE
--------------------------------------------------------

User Sends:

0.002 ETH

ETH Price:

1 ETH = $2000


Calculation:

(2000 × 0.002) = 4 USD


Solidity Representation:

4 × 10^18


So the function returns:

4000000000000000000


Which represents:

$4


--------------------------------------------------------
MENTAL MODEL
--------------------------------------------------------

Think of it like this:

Multiply → Add decimal precision
Divide → Remove extra decimal precision


Solidity Pattern:

Oracle Data → Convert to 18 decimals
Perform Math Operations
Normalize back using division
